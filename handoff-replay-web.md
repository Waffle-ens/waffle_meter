# 핸드오프 — 통계웹 리플레이 재생 (WCL식 이동 리플레이)

> 대상: 통계웹 레포 담당 에이전트/개발자.
> 미터(이 레포 + 비공개 엔진 레포)는 전투별 **위치 리플레이 레코딩**을 이미 생성·저장한다.
> 이 문서는 웹이 그 레코딩을 재생하는 데 필요한 **전체 계약**이다: 와이어 포맷, 재생 알고리즘,
> 던전 지도 투영, 업로드 경로 제안. 웹 외의 지식(패킷 RE 등)은 필요 없다.

---

## 1. 데이터: `ReplayRecording` JSON (schema 1)

미터가 전투 종료(킬 **그리고** 전멸/직전 전투 모두)마다 1개 생성. 로컬은
`%APPDATA%\waffle_meter.v1.4\replays\replay-{startMs}.json`으로 저장된다. 직렬화 소스 오브 트루스:
`dotnet/src/WaffleMeter.Replay.Abstractions/ReplaySerializer.cs` (공개 레포 — 웹 계약은 여기만 보면 됨).

```jsonc
{
  "schema": 1,               // 포맷 버전. 1이 아닌 값 = 재생 거부(향후 확장)
  "epoch": 0,                // 미터 내부 리셋 epoch (웹은 무시 가능)
  "startMs": 1782212099677,  // 전투 시작 절대 wall-clock (ms). 트랙 포인트 t의 기준점
  "endMs": 1782212235140,    // 전투 종료 절대 wall-clock (ms)
  "bossDefeated": false,     // true=킬 클리어, false=전멸/직전 전투(보스 생존)
  "targetCode": 2301059,     // 보스 mob 코드 (없으면 필드 자체가 생략)
  "targetName": "무스펠",     // 보스 이름 (없으면 생략)
  "tracks": [
    {
      "uid": 13194,          // 엔티티 uid (데미지 액터 id와 같은 키스페이스)
      "nick": "에젤",         // 닉네임 ("" 가능 — 미해소 무버)
      "srv": 2003,           // 서버 id
      "job": "호법성",        // 직업명 로컬라이즈드 ("" 가능)
      "self": false,         // 미터 사용자 본인
      "target": false,       // 보스/타깃 트랙
      "slot": 0,             // 8인 공대 서브파티 슬롯 1-8 (1-4=1파티, 5-8=2파티), 0=미상
      "op": "3729",          // (진단용) 디코딩된 opcode hex — 웹은 무시
      "off": 2,              // (진단용) — 웹은 무시
      "pts": [               // 경로 포인트, t 오름차순
        [998, 19316.0, 57324.3, -3414.0],   // [t(ms, startMs 기준), 월드X, 월드Y, 월드Z(높이)]
        [1398, 19289.4, 57094.0, -3419.0]
      ]
    }
  ]
}
```

핵심 시맨틱스:
- **포인트는 수신된 그대로(희소·불규칙 케이던스)** 저장된다. 30fps는 재생 시 보간으로 만든다
  (§2) — 저장 비용 0 (WCL 모델).
- `pts`가 **빈 배열인 트랙도 정상**: 전투에 참여했지만 위치 방송이 안 잡힌 참가자
  (AoI/관심영역 갭). UI는 로스터에 표시하되 "위치 데이터 없음" 처리.
- 파티/공대 스코핑·신원 동결(엔티티 id 재사용 방어)은 **미터가 이미 처리**했다. 웹은 tracks를
  그대로 신뢰하면 된다.
- 크기: 8인×수 분 전투 ≈ 수백 KB(비압축 JSON). gzip 전송 시 ~1/6.

## 2. 재생 알고리즘 (인미터 플레이어와 동일하게)

레퍼런스 구현: `dotnet/src/WaffleMeter.App.Wpf/ReplayWindow.xaml.cs` (공개 레포).

1. **타임라인**: 재생 시각 `ms ∈ [0, endMs-startMs]`. 30fps 틱(33ms)마다 각 트랙의 위치 산출.
2. **보간**: 플레이헤드를 둘러싼 두 포인트 `p[i], p[i+1]` 사이 선형 보간.
   단, **두 포인트 간격이 1500ms를 넘으면 보간하지 말고 이전 포인트에 HOLD**(정지 표시).
   진짜 데이터 단절(사거리 이탈/텔레포트)을 미끄러지듯 이동하는 가짜 모션으로 그리면 안 된다.
   마지막 포인트에서 1500ms 지나면 stale 처리(점 흐리게).
3. **트레일**: 플레이헤드에서 끝나는 최근 연속 구간(갭 1500ms 초과 시 절단)을 짧은 꼬리로.
4. **컨트롤**: ▶/❚❚, 스크럽 슬라이더(0..duration, 드래그 즉시 시킹), 배속 0.5~4x.
5. **표기**: 직업색 점, self 강조(금), 보스 구분(적). 호버 툴팁 `nick (job) · (Z m)` —
   높이는 호버에서만. 겹친 점 클릭 → 겹친 캐릭터별 높이 리스트 팝업.

## 3. 던전 지도 배경 + 좌표 투영

지도 카탈로그: `dotnet/Assets/Maps/map-manifest.json` + PNG 9종 (공개 레포, 총 ~9MB).
레퍼런스 구현: `dotnet/src/WaffleMeter.App.Wpf/ReplayMapCatalog.cs`.

- 매칭: `recording.targetCode ∈ entry.bossCodes` 인 엔트리 선택. 없으면 **상대 플롯 폴백**
  (모든 포인트의 bbox에 패딩 주고 화면 fit — 지도 없이도 가치 80%).
- 투영 (월드 → 이미지 픽셀):
  ```
  u = (X - worldMinX) / (worldMaxX - worldMinX) * imageWidth
  v = (Y - worldMinY) / (worldMaxY - worldMinY) * imageHeight   // 월드 +Y = 이미지 아래방향
  ```
- 뷰는 액션에 줌: 레코딩 bbox(패딩)로 클램프. 지도 전체를 다 보여줄 필요 없음.

## 4. 업로드 경로 (미정 — 사장님 결정 필요, 제안 포함)

현재 상태: **미터→웹 리플레이 업로드는 아직 배선되지 않았다.** 로컬 저장 + 인미터 재생만 동작.

제안 (플랜 §6b, 권장은 A):
- **(A. 권장) 옵트인 공유**: 인미터 리플레이 창에 "웹에 공유" 버튼 → 해당 레코딩 1건 업로드 →
  공유 URL 반환. 킬/wipe 무관하게 사용자가 고른 것만 올라감. 리더보드 오염 없음, 놀람 최소.
- **(B) 킬 업로드 동반**: 기존 전투 업로드 페이로드에 optional `replay` 필드 추가. 자동이지만
  페이로드가 수백 KB 커지고, 파티원 위치까지 자동 공개되는 동의 문제가 생김.

웹 측 준비물(A 기준):
- `POST /api/replays` — body = 위 JSON(gzip), 응답 = 공유 id/URL. **보안앵커 서명 필수**
  (install ECDSA 서명 — 기존 report/consent write와 동일한 §2.1 서명 스킴을 그대로 탄다.
  미터측 서명 유틸 재사용 가능: `StatsApiClient` 경유).
- 조회 `GET /api/replays/:id` + 재생 페이지(§2 알고리즘). 20/30인도 점 N개 렌더일 뿐이라 캔버스로 충분.
- 사이즈 가드(예: 2MB 상한), rate limit, 보존 기간 정책.
- **프라이버시**: 리플레이에는 파티원 닉/서버/직업/위치가 들어있다. 공유는 업로더의 명시 행동
  (A안)이며, 대상 전투의 파티원 비공개 설정과의 상호작용은 웹 정책으로 결정할 것.

미터측 배선 지점(웹 API 확정 후): `ReplayWindow`에 공유 버튼 → `StatsApiClient`로 POST.
전투 업로드 동반(B안)이면 `StatsUploadQueue.OfferIfEligible` / `StatsPayloadBuilder`에 필드 추가.

## 5. 웹 구현 검증용 샘플

- 실코퍼스에서 뽑은 샘플 JSON: 미터 레포의 `ReplayGenCli`(비공개 시블링 레포 도구)가 생성 —
  요청하면 몇 개 전달 가능. 필드 커버리지: bossDefeated true/false, slot 1-8, 빈 pts 트랙,
  "" nick 포함.
- 라운드트립 파서 테스트 벡터는 `ReplaySerializerTests.cs`(공개 레포) 참고.

## 6. 남은 미결(웹과 무관하게 미터쪽에서 진행 중)

- **self(본인) 트랙은 희소하거나 거의 비어 있을 수 있음 — 근본 한계로 확정.** self의 밀집 이동은
  outbound(클라→서버) 스트림에만 있고 **암호화**돼 캡처로 복원 불가. inbound은 self 위치를 안 돌려줌
  (MMO 권위모델). self는 희소 inbound 키프레임(0x3746 높이 + 간헐 0x373C)만 가짐. → 웹 재생은 §2의
  **HOLD 규칙**만 지키면 자연 처리(빈 pts면 로스터에 "위치 없음"). 파티/공대원(WCL 핵심가치)은 조밀·무관.
- 대형(20/30인) 레이드의 AoI 완결성은 라이브 검증 중(8인은 99% 확인) — 빈 pts 트랙 처리만 해두면 영향 없음.
